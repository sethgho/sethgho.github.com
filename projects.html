MENU LINE GEORGE

sethgholson.com/george

UPDATE: George has retired :(

Unfortunately, Christus Schumpert St. Mary has closed. As a result, George has retired, from what I understand. This means I'm shuttering this little app (until I happen across another entertaining preacher reading daily hospital cafeteria menus). If you're interested in seeing the old app, though, you can try it out in the browser.

My home town of Shreveport, LA has a very unique hospital cafeteria. Christus Schumpert Hospital's daily phone-in cafeteria menu is nearly always narrated by Cafeteria Team Leader George Cauthen. This isn't your ordinary hospital cafeteria menu, though. George adds a southern evangelical flair to his recording that must be heard to understand. I was first introduced to this when I worked in IT for a clinic on the Christus Schumpert campus. George has dished out some fantastic sound bytes over the years, but once the day is over they're gone forever. Needing a quick and simple project to explore iOS development I could complete while on paternity leave (because newborns are a breeze, right?), preserving these calls seemed like a great little project. The three main hurdles were:

  • Calling
  • Managing
  • Apps

CALLING AND RECORDING

After a few different iterations using a couple of different languages and APIS, I'm currently running a node.js app on a Raspberry Pi employing Twilio to initiate and record the calls. Twilio lets me download the file in MP3 format, which is passed up to Parse.

ADMINISTRATION

Because I'm using Parse for my back end, I require no server side code. Using their Javascript SDK, I created a web app hosted on Github Pages to manage these recording. Backbone.js allowed me to us a simple MVC pattern. Bootstrap gave me a grid system and some halfway decent standardized controls.

CLIENTS

My first verison of the iOS app was created using the Titanium Mobile. Using a cross platform framework like this was a wrong decision. Versions 2.x and 3.x were rewritten in Objective-C.

The Android client is targeting 2.3.x (Gingerbread)+ using the native Java SDK. Currently still in development, it should be completed and available in Google Play relatively soon.

Though not a full client, I've also added a simple web based player for sharing. This is useful for generating links via the sharing mechanisms in the mobile app.
